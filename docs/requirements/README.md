## 中文

把后续需求说明书放在这个目录下。

建议：

- 每个需求说明书单独一个 Markdown 文件
- 文件名稳定，例如 `phase1-requirements.md`
- 如果需要核对实现状态，尽量使用可枚举条目

后续我会基于这里的文件，逐项核对当前功能是否实现、是否达标。

## English

Place future requirement specifications in this directory.

Recommended conventions:

- One Markdown file per requirement set
- Stable filenames, for example `phase1-requirements.md`
- Prefer explicit checklist-style items for implementation validation

I will use the files here to verify feature coverage and acceptance status item by item.
